Main Methods for Showing Results#

Deepchecks Check and Suite results can be viewed in a separate window or inline if working within a notebook.

show() method#

This is the main show method. It will show the check/suite resultâ€™s HTML output inline, in a notebook cell.

result.show()

show_in_window() method#

This method shows the result in a separate window (using the PyQt5 library). It starts a PyQt5 application in a separate process therefore this display method does not work on cloud platforms like Kaggle or Google Colab.

result.show_in_window()

Note, PyQt5 library is an optional dependency and it is not installed along with Deepchecks automatically, therefore to use show_in_window() you need to install it manually by running the next command:

$ pip install PyQt5 pygtwebengine

Alternative Methods for Showing Results (Troubleshooting)#

The resultâ€™s HTML output produced by the show() method requires specific jupyter extensions to display properly. These extensions may not be installed/enabled by default, or may have compatibility issues with different environments (e.g. some cloud platforms).

Therefore, if you experience any result display problems, you can consider any of the following options to resolve them:

  • Using one of the alternatives: show_in_iframe() method or show_not_interactive() method, elaborated below.

  • Verifying that the jupyter-widgets extension is installed and enabled.
    If youâ€™re working on jupyterlab, same for jupyterlab-plotly extension.
    Restart your jupter server after these changes so that they will have an effect.
    For more information and instructions take a look at a plotly JupyterLab Troubleshooting
    documentation section.

  • Exporting the results (e.g. saving them to html) and viewing the output file.

This is the main alternative in case you experience any result display problems. It uses a different output technique (embedding the resultâ€™s HTML output into an iframe) and thus doesnâ€™t require any jupyter extensions installed.
